# 基于 i.MX6NULL 的 Linux 驱动开发与多媒体应用平台

> 本项目是以 I.MX6ULL 为硬件平台，基于 Qt 搭建的用于 Linux 各类设备驱动及应用的综合测试平台，包括实现 GPIO、I2C、SPI、ADC 等驱动统一调试界面，并通过虚拟设备模拟驱动接口实现跨平台运行

## 硬件
- UVC 相机 
- 画图板
- 姿态角展示
- 环境光监测 
- ADC 采集
- 设置 
- CAN 接收测试
- UART 接收测试
- 音乐播放器
- 视频播放器

## 实现功能
- CAN通信测试
  - 接收从机消息
  - 单机Send按钮发消息给从机
- 画图板
  - 支持任意颜色绘制
  - 鼠标状态下支持滚轮调节线条粗细
  - 单击保存按键可以将绘制的图像保存到本地
  - 橡皮擦除线条
  - 单击扫把按键可以一键清除画布绘制内容
  - 单击相册可以查看保存的画布图像，支持长按放大显示
- 支持UVC相机
  - 单击打开按键打开相机
  - 单击拍照即可拍摄照片
  - 单击相册查看保存相片
  - 支持长按放大显示
- 接触测试和环境光监测
  - 利用 AP3216C 
- ADC
  - 可采集 GP101 1001引脚输入的电压
  - 单击图例可显示/隐藏曲线
- 支持UART通信测试
  - (目前仅有CH34x驱动)
- 音乐播放器
  - 可从文件管理器选择要播放的音乐
  - 支持拖动进度条改变歌曲进度
  - 可以调节喇叭音量大小，修改完毕后两秒后自动隐藏
  - 支持循环、乱序、单曲循环三种播放模式
  - 支持黑胶唱片播放、暂停动态效果
- GPIO控制外设
  - 单击LED 图标打开/关闭开发板上的LED 灯
  - 单击 Beep图标打开/关闭开发板上的蜂鸣器
- 支持调节屏幕亮度
- 视频播放
  - MP4格式文件

## 项目介绍

### 源码结构

- IMx6ULL.pro
  - 项目管理文件，这个项目管理文件十分重要，当您加入了文件或者删除了文件， Qt Creator 会自动修改这个*.pro 文件。*
  - 有时候需要打开这个.pro 文件添加我们的设置项    
- Headers目录
  - 存放的是项目内所有的头文件*.h    
- Sources目录
  - 存放的是项目内的所有 C++源码文件*.cpp    
- Forms目录
  - 放项目内所有界面文件*.ui。
  - *.ui 文件由 XML 语言描述组成，编译时会生成相应的 cpp 文件，这样交叉编译器就可以编译它了    
- Resources目录

## 复现
1. **启动阶段 (main.cpp)**：内核加载完成后，执行二进制文件。`main.cpp` 创建 `QApplication` 对象，并实例化 `MainWindow`。
2. **主界面呈现 (mainwindow.cpp)**：屏幕上显示出各种功能的图标（音乐、传感器、设置等）。
3. **功能分发**：
   1. 当你点击“GPIO”图标：`MainWindow` 会隐藏自己（或作为父窗口），创建 gpio 类的实例。
   2. **数据采集循环**：在 gpio.cpp` 中，控制 LED 亮灭本质上就是向 /sys/class/leds/xxx/brightness 文件写入 "1" 或 "0"。
4. **硬件控制（若有）**：
   1. 如果你在 UI 上点亮 LED：`gpio.cpp` 会被调用，它打开 `/sys/class/gpio/gpioX/value` 并写入 "1"

# 升级
通过软件模拟方式实现在缺少硬件的情况下，实现对应实验，使得底层理解更加深入。
| **模块类型**         | **代表硬件**                            | **核心组件/协议**                         | **软件模拟方案 (PC 端)**                                     | **模拟重点 / 关键技术**                                      |
| -------------------- | --------------------------------------- | ----------------------------------------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
| **基础输入输出**     | **GPIO** (LED/蜂鸣器/按键)              | `sysfs` 接口, `/sys/class/gpio`           | **文件读写模拟**：在 PC 创建同名目录，或用 `printf`/`QLabel` 颜色变化替代硬件动作。 | 状态同步、中断模拟（用 `QSocketNotifier` 监控文件变化）。    |
| **低速通信接口**     | **UART / CAN**                          | `/dev/ttyS*`, `SocketCAN`                 | **虚拟串口对 (socat)**：使用 `socat` 创建成对的虚拟串口；CAN 可使用内核自带的 `vcan` 驱动。 | 数据帧的打包与解析、校验位计算、异步收发逻辑。               |
| **传感器 (I2C/SPI)** | **AP3216C** (光强), **ICM20608** (六轴) | `ioctl`, `/dev/i2c-*`                     | **数据序列模拟**：使用随机数或预设的 JSON/CSV 数据文件循环读取，模拟数值波动。 | 寄存器读写逻辑的封装、数据转换公式（如原始数据转为 lux 或 g）。 |
| **多媒体音频**       | **Music Player**                        | `QProcess`, `MPlayer`, `ALSA`             | **跨平台后端重定向**：PC 端同样安装 `mplayer`，仅需修改音乐文件路径和系统音量控制命令。 | 进程间通信 (Slave Mode)、UI 动画同步（黑胶转动、进度条更新）。 |
| **多媒体视频**       | **Video Player**                        | `QProcess`, `MPlayer`, `V4L2`             | **软件缩放模拟**：MPlayer 加入 `-softvol`，视频输出改用 `x11` 或 `xv` 而非 `fbdev`。 | 指令刷新 (flush)、UI 层级遮挡处理 (raise)、窗口句柄 (`WId`) 嵌入。 |
| **实时图像采集**     | **Camera**                              | `V4L2`, `ioctl`, `mmap`                   | **1. 内核级**：`v4l2loopback` 推流。 **2. 应用级**：`#ifdef` 绕过 ioctl，直接用 `QPixmap` 加载图片。 | 帧率控制 (Timer)、内存映射逻辑模拟、图像格式转换 (MJPEG/YUYV)。 |
| **模拟信号采集**     | **ADC** (电位器/电池电压)               | `IIO` (Industrial I/O), `/sys/bus/iio`    | **正弦波/随机数模拟**：使用 `QTimer` 配合 `qSin()` 函数产生周期性波形，模拟真实电压波动。 | 采样频率模拟、数据量纲转换（12位 ADC 的 0-4095 转为 0-3.3V）。 |
| **参数管理**         | **Config** (系统设置/校准)              | backlight 驱动节点，/sys/devices/platform | **本地文件映射**：将 路径重定向到 /tmp 目录下的模拟文件。    | 与 GPIO 的模拟逻辑完全一致。                                 |
